# NS-MeetAI 架构设计文档

## 🏗️ 系统架构概述

NS-MeetAI 采用微服务架构，分为三个核心服务：

1. **前端服务** - 用户界面和交互
2. **后端服务** - 业务逻辑和API网关
3. **AI处理服务** - AI模型和数据处理

## 🔧 技术栈选择

### 前端技术栈
- **框架**: React 18 + TypeScript
- **样式**: Tailwind CSS + Headless UI
- **状态管理**: Zustand (轻量级状态管理)
- **路由**: React Router v6
- **HTTP客户端**: Axios
- **构建工具**: Vite
- **测试**: Vitest + React Testing Library

### 后端技术栈
- **运行时**: Node.js 18+
- **框架**: Express.js + TypeScript
- **ORM**: Prisma (数据库操作)
- **认证**: JWT + Passport.js
- **验证**: Zod (数据验证)
- **日志**: Winston + Morgan
- **测试**: Jest + Supertest

### AI处理技术栈
- **语音转文字**: OpenAI Whisper API
- **文本处理**: OpenAI GPT-4 API
- **NLP库**: Natural / Compromise
- **队列处理**: Bull (Redis队列)
- **向量存储**: Pinecone (可选)

### 数据存储
- **主数据库**: PostgreSQL (生产) / SQLite (开发)
- **缓存**: Redis
- **文件存储**: MinIO / AWS S3
- **搜索**: Elasticsearch (可选)

## 🚀 部署架构

### 开发环境
```
本地机器 → Docker Compose (所有服务)
```

### 生产环境
```
前端 (Vercel/Netlify) → 后端 (Railway/Render) → AI服务 (单独部署)
        ↓                    ↓                    ↓
    CDN缓存             负载均衡器             消息队列
        ↓                    ↓                    ↓
    用户访问             业务处理             AI处理流水线
```

## 🔄 数据流设计

### 会议处理流程
```
1. 用户上传音频文件
   ↓
2. 后端验证并存储文件
   ↓
3. 发送到AI处理队列
   ↓
4. AI服务处理:
   - 语音转文字 (Whisper)
   - 文本清洗和分段
   - 关键信息提取
   - 任务识别
   - 情绪分析
   ↓
5. 生成结构化会议纪要
   ↓
6. 存储到数据库
   ↓
7. 返回结果给用户
   ↓
8. 用户可编辑和导出
```

### 实时处理流程
```
1. 用户开始录音
   ↓
2. WebSocket连接建立
   ↓
3. 实时音频流传输
   ↓
4. 流式语音识别
   ↓
5. 实时文本显示
   ↓
6. 实时摘要更新
   ↓
7. 会议结束生成完整纪要
```
